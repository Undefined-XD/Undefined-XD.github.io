Spring 学习（未完待续）

一、IOC

  1. 在 Maven 中引入 Spring 框架
  1. 在 resources 目录下创建 bean.xml
  1. 把创建对象交给 spring 来管理，在需要用到的地方获取 bean

Bean 配置创建对象的三种方法

  1. 直接指定类创建
  1. 通过工厂类的普通方法创建
  1. 通过静态方法创建

Bean 的作用范围调整

通过 Bean 标签的 scope 属性设置：

  1. singleton：单例（默认）
  2. prototype：多例
  3. request：Web应用请求
  4. session：Web应用会话
  5. global-session：集群环境的作用范围，当不是集群时就是 session

Bean 的生命周期

  1. 单例对象

    • 出生：当容器创建时对象出生
    • 活着：只要容器还在，对象一直活着
    • 死亡：容器销毁，对象消亡

  2. 多例对象

    • 出生：使用对象时 spring 框架为我们创建
    • 活着：只要对象在使用中就一直活着
    • 死亡：当对象长时间不用，且没有别的对象引用时，由java垃圾回收器回收

  3. 给 bean 属性添加生命周期的方法：

    • <bean init-method=”init_mothod_name” />
    • <bean destroy-method=”destroy_mothod_name” />

Bean 注入数据

  1. 默认情况下，bean 实例化时使用的是 bean 类的默认无参构造函数，否则报错。

  2. 如果需要使用有参数的构造函数，需要进一步配置 bean 对象：

    • 构造函数注入：在 bean 内部使用 <constructor-arg/> 标签

    • set 方法注入：

      对于一个用 bean，如果设置了 property 、init-method、destroy-method、static 代码块、构造方法、bean实例函数调用，方法的执行顺序为：

      1. static 代码块
      2. constructor 构造方法
      3. set 方法
      4. method-init
      5. bean 实例函数调用
      6. method-destroy

    • 集合数据注入

二、使用反射实现注入

  1. 修改 xml 文件

  2. 对要用到的类使用 @Component 注解，用于创建对象：

    当没有对 @Component 赋值时，默认使用当前类名（首字母小写）作为 key，调用类的过程同bean形式。

  3. @Component 的衍生注解，效果同 @Component，只是应用层级更清晰：

    • @Controller：用于表现层
    • @Service：用于业务层
    • @Repository：用于持久层

  4. 在要自动注入数据的变量前使用 @AutoWired ，用于自动按照类型注入，相当于<property> 属性：

    使用过程和规则：

    1. 当 main 方法中调用 test2.addAccount() 时，在 test2 实例中 addAccount 需要用到 IAccountService test3，正常情况下会报错，因为方法执行时没有获取到 IAccountService 类型的实例。
    2. 使用 @AutoWired 注解，首先根据 IAccountService 这个数据类型，在容器中寻找是否有同类型（或者可以通过多态看成是同一个类型的）且唯一的 Bean 实例，如果有，那么就直接将容器中的实例赋值给这个变量。
    3. 如果没有同类型的 Bean 实例对象，则报错。
    4. 如果同类型 Bean 实例对象有多个，那么将根据当前变量的变量名，在容器 Map 寻找同名 key ，将 key 对应的 Bean实例赋值给这个变量。

  5. @Qualifier("指定的同类型 bean对象 id")必须结合 @AutoWired，可以具体给变量赋值，避免自动根据变量名寻找 Bean 实例赋值。

  6. @Resource(name = "指定的同类型 bean对象 id") 单独使用可以给变量赋值

  7. 注意：@AutoWired 、 @Qualifier()、 @Resouce(name="") 只能用于注入 bean 类型的数据，无法注入基本类型和 String 类型。另外，集合类型的注入只能通过 XML 实现。

  8. @Value("基本数据类型和String类型值")，可以直接赋值。里面也可以使用 EL 表达式：${表达式}

  9. @Scope("作用域") 同 XML 中的 scope 属性和取值一样，设置作用范围。

  10. @PostConstruct 修饰方法，同 XML 中的 init-method 属性，设置初始化函数。

  11. @PreDestroy 修饰方法，同 XML 中的 destroy-method 属性，设置销毁函数。

三、纯注解的方式实现注入

创建配置类

为什么需要配置类？因为我们只能在自己写的类中通过注解来实现生成 bean 实例并存入容器中。对于如 jar 包中引入的类，我们无法修改，但还是要实现纯注解方式实现注入，而不依赖于 xml 文件的话，就需要写一个配置类，才能统一使用纯注解实现。

  1. @Comfiguration 注解表示这个类是一个配置类，只有标识了这个注解的类在被扫描到的时候才会生成 bean 对象给 IOC 容器管理。

  2. @ComponentScan 注释表示创建容器需要扫描的包路径，效果相当于

  3. 对于从 jar 包中注入的类，一般都通过构造函数形式创建 bean 实例，我们可以通过在配置类中定义方法，方法返回对应的 jar 包类实例：

    • @Bean(name="bean实例名") 表示将方法返回对象存入 IOC 容器中，key 为注解指定的名字，或者默认使用方法名。
    • 对于使用了该注解的方法，如果有参数，spring 框架会自动去容器中查找有没有可用的 bean 对象。查找的方式跟 @AutoWired 是一样的。

  4. 如果要实现一个父配置类，在其中加载子配置类，可以使用 @Import(子配置类名.class, ...) 注解，加载其他配置类。被加载的子配置类可以不用 @Configuration 注解。
